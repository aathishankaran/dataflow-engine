# Dataflow Engine - Developer Guide

## Architecture
Config-driven PySpark pipeline that processes mainframe data files. Receives a config JSON (generated by config-engine) and executes: load inputs → run transformations → write outputs.

## Project Structure
```
dataflow-engine/
├── run_dataflow.py                    # CLI entry point
├── requirements.txt                   # Python 3.6.8+ dependencies
├── settings.json                      # Runtime settings (bucket prefixes, holidays)
└── dataflow_engine/
    ├── __init__.py                    # Package exports
    ├── runner.py                      # DataFlowRunner - main pipeline orchestrator
    ├── transformations.py             # PySpark transformation executor
    ├── config_loader.py               # Config JSON loading & path resolution
    ├── oracle_loader.py               # Oracle SQL*Loader integration
    ├── vault.py                       # HashiCorp Vault credential management
    └── incident.py                    # Moogsoft/ServiceNow incident connector
```

## Pipeline Execution Flow
```
CLI (run_dataflow.py)
  → argparse (config.json, --base-path, --master, --dry-run, --settings, --no-cobrix)
  → _build_spark_session() — OS-aware SparkSession (Windows/macOS/Linux)
  → DataFlowRunner(spark, config_path, base_path, use_cobrix, settings)
      → config_loader.load_config(path) — parse JSON, normalize keys
      → runner.load_inputs() — read files into Spark DataFrames (COBOL/CSV/FIXED/PARQUET/DELIMITED)
      → runner.run_transformations() — apply transformation steps via MainframeTransformationExecutor
      → runner.write_outputs() — write DataFrames to output files (with header/trailer for FIXED)
```

## Module Reference

### run_dataflow.py — CLI Entry Point
**Functions:**
- `_build_spark_session(master: str) -> SparkSession` — OS-aware session builder
  - Windows: HADOOP_HOME config, Windows-friendly scratch dir
  - macOS: snappy compression codec
  - Linux: default config
- `_error_messages(exc) -> List[str]` — Walk exception chain for clean messages
- `main()` — argparse CLI

**CLI Arguments:**
| Argument | Default | Description |
|----------|---------|-------------|
| config | samples/config.json | Path to config JSON |
| --base-path | parent of config | Base path for relative paths |
| --no-cobrix | False | Disable Cobrix COBOL reader |
| --master | local[*] | Spark master URL |
| --dry-run | False | Skip output writes |
| --settings | settings.json | Path to settings JSON |

### config_loader.py — Config Loading & Path Resolution
**Public Functions:**
- `load_config(path) -> Dict` — Load and validate config JSON. Supports PascalCase and lowercase keys. Raises FileNotFoundError, ValueError.
- `get_input_path(config_input, base_path=None, interface_name="", settings=None) -> str` — Resolve input file path. Priority: dataset_name (v2) → source_path+source_file_name (v1) → path → s3_path → cobrix → dataset
- `get_output_path(config_output, base_path=None, interface_name="", settings=None) -> str` — Resolve output path. V2: curated_bucket_prefix + frequency/date. V1: source_path + frequency/date. Legacy: path/s3_path as-is.
- `get_control_file_path(config_input, base_path=None, interface_name="", settings=None) -> str` — Resolve control/count file path
- `get_count_file_path(config_input, base_path=None) -> str` — Alias for get_control_file_path
- `has_count_file(config_input) -> bool` — Check if input has companion count file
- `get_frequency(config, node_name) -> Optional[str]` — Get frequency for a node
- `get_previous_business_day(target_date, holidays) -> date` — Previous business day (skips configured holidays only, NOT weekends)
- `get_input_path_for_date(config_input, partition_date, ...) -> str` — Input path for specific date
- `get_validate_paths(logic, ...) -> Tuple[str, str, str, str]` — Validation output paths

**Private helpers:**
- `_resolve_partition_col(expr) -> str` — Resolve date expressions: load_date(), current_date(), date_sub(), date_format(), last_day()
- `_build_partitioned_path(source_path, source_file, frequency, partition_col) -> str` — Compose partitioned path: <path>/<FREQUENCY>/<YYYYMMDD>/<file>

### runner.py — DataFlowRunner (Main Pipeline)
**Class: DataFlowRunner**
- `__init__(spark, config_path, base_path, use_cobrix=True, settings=None)` — Load config, initialize registries
  - Attributes: spark, config, input_dfs (Dict[str, DataFrame]), output_dfs (Dict[str, DataFrame])
- `load_inputs()` — Read all input files into DataFrames
  - Supports: COBOL (Cobrix), CSV, FIXED (fixed-width), PARQUET, DELIMITED
  - FIXED format: reads as single column, then substr() for each field
  - Handles header_count/trailer_count skipping
- `run_transformations() -> Dict[str, DataFrame]` — Execute all transformation steps in order
  - Returns output_dfs mapping
- `write_outputs()` / `_write_outputs()` — Write DataFrames to files
  - FIXED format: generates fixed-width text with header/trailer rows
  - CSV/PARQUET/DELIMITED: standard Spark write
  - Handles output_columns selection, coalesce, ctrl file generation

### transformations.py — Transformation Executor
**Class: MainframeTransformationExecutor**
**Public function: `apply_transformation_step(step, input_dfs, spark, ...)`**

**Transformation Types:**
| Type | Description | Key Logic Fields |
|------|-------------|-----------------|
| filter | Row filtering | conditions: [{field, operation, value}] |
| select | Column projection OR expression computation | columns (projection) OR expressions (withColumn) — MUTUALLY EXCLUSIVE |
| join | DataFrame join | left, right, on: [[left_col, right_col]], how |
| aggregate | Group by + agg | group_by, aggregations: [{field, operation, alias, condition}] |
| union | Union DataFrames | source_inputs |
| validate | Data validation | rules, mode (FLAG/DROP/ABORT), ctrl file generation, prev-day checks |
| custom | Sort/merge | operation: "sort", key |
| oracle_write | Oracle SQL*Loader | host, port, service_name, schema, table, vault_path |

**Important: select step behavior:**
- If `logic.columns` is provided → `.select()` projection and RETURN (expressions are IGNORED)
- If `logic.expressions` is provided (without columns) → `.withColumn()` for each expression
- These are mutually exclusive by design

### oracle_loader.py — Oracle SQL*Loader Integration
**Public Functions:**
- `write_df_to_oracle(df, logic, pipeline_name="", step_id="")` — End-to-end Oracle load: Vault creds → CSV export → CTL generation → sqlldr subprocess
- `generate_ctl_content(table_name, schema, columns, spark_types, load_mode="APPEND", bad_file, log_file, discard_file=None, batch_size=10000) -> str` — Generate SQL*Loader .ctl file content

**Type Mapping (Spark → SQL*Loader):**
| Spark Type | SQL*Loader Type |
|-----------|----------------|
| string | CHAR |
| integer/long | INTEGER EXTERNAL |
| double/float/decimal | DECIMAL EXTERNAL |
| boolean | CHAR |
| date | DATE "YYYY-MM-DD" |
| timestamp | TIMESTAMP "YYYY-MM-DD HH24:MI:SS" |
| binary | RAW |

**Load Modes:** INSERT, APPEND, TRUNCATE (from TRUNCATE_INSERT), REPLACE

### vault.py — HashiCorp Vault Integration
**Public Functions:**
- `get_oracle_credentials(vault_path, username_key="username", password_key="password") -> Tuple[str, str]` — Read Oracle creds from Vault KV (tries v2 then v1)
- `store_oracle_credentials(vault_path, username, password, extra_fields=None)` — Write creds to Vault KV v2

**Authentication (in order):**
1. VAULT_TOKEN env var (dev/direct token)
2. VAULT_ROLE_ID + VAULT_SECRET_ID (AppRole - production)

**Environment variables:** VAULT_ADDR, VAULT_TOKEN, VAULT_ROLE_ID, VAULT_SECRET_ID, VAULT_NAMESPACE, VAULT_SKIP_VERIFY

### incident.py — Moogsoft/ServiceNow Incident Connector
**Class: MoogsoftIncidentConnector**
- `__init__(endpoint, api_key, timeout=30, default_source="dataflow-engine", verify_ssl=True)`
- `create_incident(summary, description, severity, source, alert_key, tags, service, additional_fields) -> str` — Generic incident creation
- `create_validation_incident(pipeline_name, step_id, invalid_count, ...) -> str` — Data validation failure
- `create_last_run_file_missing_incident(pipeline_name, step_id, file_path, ...) -> str` — Last run date file missing
- `create_input_file_missing_incident(pipeline_name, input_name, file_path, ...) -> str` — Input file not found
- `create_prev_day_check_incident(pipeline_name, input_name, file_path, expected_date, actual_date, ...) -> str` — Previous day header mismatch
- `create_record_count_check_incident(pipeline_name, step_id, input_file_path, expected_count, actual_count, ...) -> str` — Trailer count mismatch

**Severity levels:** CRITICAL (5), HIGH (4), MEDIUM (3), LOW (2), INFO (1)

**Class: IncidentCreationError(RuntimeError)** — Raised on API failures

## Key Design Decisions
1. **Previous day calculation**: `get_previous_business_day()` only skips configured holidays from settings. Weekends are NOT skipped — they are valid business days in this system.
2. **Select columns/expressions**: Mutually exclusive. `columns` does .select() and returns immediately. `expressions` does .withColumn() for each.
3. **Control files**: Written to `ctrl/<step_id>/<frequency>/<date>/<name>.CTL` with record counts and validation metadata.
4. **V1 vs V2 config schema**: V1 uses source_path + source_file_name. V2 uses dataset_name with bucket prefixes from settings.
5. **Fixed-width I/O**: Reads entire line as single string column, then uses substr() per field. Writes using format_string() with right/left padding.
